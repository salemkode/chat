KNOWN RISKS
================================================================================

RISK-1: Memory Extraction Accuracy
  Description: LLM-based fact extraction may be <80% accurate
  Likelihood: MEDIUM
  Impact: HIGH
  Mitigation:
    - Use GPT-4 for extraction (higher accuracy than GPT-3.5)
    - Implement manual curation tool (allow users to edit/delete memories)
    - Monitor extraction accuracy with weekly manual reviews (100 samples)
  Status: Monitoring required in Week 5
  Owner: Memory Manager Agent developer

RISK-2: Context Build Latency at Scale
  Description: Vector search + memory ranking may exceed 200ms at scale
  Likelihood: MEDIUM
  Impact: HIGH
  Mitigation:
    - Implement context caching with 5-minute TTL
    - Optimize vector search with pre-filtering (userId, scope, scopeId)
    - Use Convex vector index (optimized for performance)
    - Load test with 10k memories in Week 5
  Status: Performance testing required in Week 5
  Owner: Context Builder Agent developer

RISK-3: Sidebar Performance Degradation
  Description: Rendering 100+ projects may cause UI lag
  Likelihood: LOW
  Impact: MEDIUM
  Mitigation:
    - Implement virtualization (react-window) early in Week 5
    - Profile render performance with 500 projects
    - Optimize re-renders with React.memo and useMemo
  Status: Implementation required in Week 5
  Owner: Frontend developer

RISK-4: User Confusion About @Mention Feature
  Description: Users may not understand how @mentions trigger context injection
  Likelihood: MEDIUM
  Impact: MEDIUM
  Mitigation:
    - In-app tooltips on first use
    - Video tutorial (2-3 minutes)
    - Clear banner messaging ("Attach this thread to [ProjectName]?")
    - User guide published before rollout
  Status: Documentation required in Week 6
  Owner: Product/UX lead

RISK-5: Data Migration Failure
  Description: Migrating existing chats to threads may cause data loss
  Likelihood: LOW
  Impact: CRITICAL
  Mitigation:
    - Test migration on staging data (10k+ chats)
    - Write rollback script
    - Verify message counts pre/post migration
    - Run migration during low-traffic window (Sunday 2-4am)
    - Monitor error logs during migration
  Status: Testing required in Week 2
  Owner: Backend infrastructure lead

TECHNICAL DEBT
================================================================================

DEBT-1: Memory Summarization Not Implemented
  Description: Old memories not summarized (may cause token bloat)
  Impact: Medium (affects context size for long-lived projects)
  Plan: Defer to Phase 2 (not critical for MVP)
  Created: 2026-01-21

DEBT-2: Basic Cosine Similarity (No Learned Re-Ranking)
  Description: Vector search uses basic cosine similarity, no ML re-ranking
  Impact: Low (accuracy may be suboptimal but acceptable)
  Plan: Defer to Phase 3 (requires ML infrastructure)
  Created: 2026-01-21

DEBT-3: @Mention Autocomplete Does Not Support Fuzzy Matching
  Description: Autocomplete uses simple substring matching, not fuzzy
  Impact: Low (minor UX annoyance)
  Plan: Implement in Week 5 if time permits, else defer to Phase 2
  Created: 2026-01-21

DEBT-4: Sidebar Does Not Support Custom Project Ordering
  Description: Projects sorted by lastActiveAt, user cannot reorder
  Impact: Low (acceptable for MVP)
  Plan: Defer to Phase 2 (requires drag-and-drop + server-side ordering)
  Created: 2026-01-21

DEBT-5: No Cross-Project Memory Retrieval
  Description: Memories scoped to single project, no cross-project search
  Impact: Medium (limits flexibility for related projects)
  Plan: Defer to Phase 2 (requires permission model)
  Created: 2026-01-21
