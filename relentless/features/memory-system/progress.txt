## Learnings

1. **Vector Search Filters:** Convex vector search filter API is limited. Workaround: fetch candidates via regular queries first, then calculate cosine similarity in-memory.

2. **TDD Approach:** Writing tests first helps clarify requirements and ensures code quality. All memory system tests passing.

3. **Retry Logic:** Exponential backoff with 3 retries works well for OpenAI API calls.

4. **Ranking Formula:** The weighted combination (0.5 relevance, 0.3 recency, 0.2 importance) provides balanced results prioritizing semantic similarity while rewarding recent activity.
